🍀 JS Execution Context (실행 컨텍스트) 란? => 실행할 코드에 제공할 환경 정보를 모아놓은 객체 자바스크립트 엔진은 코드를 실행하기 위해서 실행에 필요한 여러가지 정보들을 알고 있어야 한다. => scope, hoisting, this, function, closure 등의 동작원리를 담고 있는 자바스크립트의 핵심원리 실행 컨텍스트 => JS 코드가 실행되는 환경 모든 JS 코드는 실행 컨텍스트 내부에서 실행된다고 생각하면 된다. 함수 실행 -> 함수 실행에 해당하는 실행 컨텍스트가 생성 -> 자바스크립트 엔진에 있는 콜 스택에 쌓임 🍀 실행 컨텍스트는 언제 생성될까? - 자바스크립트 엔진이 스크립트를 처음 마주할 때 전역 컨텍스트 생성, 콜 스택에 push - 엔진이 스크립트를 읽어가면서 함수 호출을 발견할 때마다 함수의 실행 컨텍스트를 스택에 push * 함수를 선언할 때가 아닌 실행할 때 함수 실행 컨텍스트 생성 - 실행 컨텍스트 만드는 방법 1. 전역 공간 2. 함수 3. eval() 함수 => 속도나 보안이 좋지 않아 현재는 거의 쓰이지 않음 🍀 실행 컨텍스트의 종류 - 전역 실행 컨텍스트 => 모든 스크립트 코드는 전역 실행 컨텍스트 안에서 실행 => 브라우저는 window 객체, Node.js는 global 객체가 전역 실행 컨텍스트 - 함수 실행 컨텍스트 => 함수 내에 존재하는 코드 => 함수가 실행될 때마다 만들어지는 실행 컨텍스트 => 각 함수는 고유의 실행 컨텍스트를 가지며, 함수가 실행되거나 call될 때만 생성 - eval() 실행 컨텍스트 => eval 함수로 실행되는 코드 🍀 실행 컨텍스트의 구조 LexicalEnvironment 컴포넌트 / ariableEnvironment 컴포넌트 * LexicalEnvironment - 처음에는 VariableEnvironment와 같다. - 변경 사항이 실시간으로 반영된다. * VariableEnvironment - 현재 컨텍스트 내의 식별자(변수)들에 대한 정보를 가진다. - 외부 환경 정보를 가진다. - 선언 시점의 LexicalEnvironment의 스냅샷을 유지한다. 변경사항을 반영하지 않는다. 실행 컨텍스트를 생성할 때 VariableEnvironment에 정보를 먼저 담은 다음, 이를 복사해서 LexicalEnvironment를 만든다. 주로 활용하는 것은 LexicalEnvironment => VariableEnviroment는 스냅샷 유지를 목적으로 사용!! - 함수 코드 평가 순서 1. 함수 실행 컨텍스트 생성 2. 함수 LexicalEnvironment 생성 2-1. 함수 environmentRecord 생성 2-2.
